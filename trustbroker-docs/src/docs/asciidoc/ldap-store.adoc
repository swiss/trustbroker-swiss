= LDAP Store

The LDAP store is a component in the open-source to retrieve attributes from an LDAP server.

It allows chained queries so that an attribute from any previous query can be used in the following one.

If no claim selection are configured in the IDMQuery for the LDAP Store, all attributes will be fetched.

Known limitations:

- this plugin can only connect to one LDAP server

=== Chained query
To use chained query configure the LDAP's "AppFilter" with the claim of a previous IDM query (example: `IDM:<query_name>:<definition_name>` as in `IDM:IDENTITY:uid`).



== Identity Selection

When the LDAP plugin retrieves multiple identities from the store, an Identity Selection can be triggered.

Additional information can be displayed to the user, to help choosing the Identity.
Any additional information can be used (most probably LDAP attributes).

=== Parameters
In the SetupRP, section ``ProfileSelection``.
[cols="2,3,1,1,2"]
|===
| Parameter | Meaning | Type | Values | Example

| enabled
| enables the Profile Selection
| Boolean
| true / false
| true

| handler
| needs to be explicitly specified to pick the LDAP Identity Selection handler
| String
| LDAP
| LDAP

| profileSelector
| claim to look for multiple identities. Has to be an LDAP claim, and unique.
| String
| mail, uid, ..
| mail

| additionalInfo
| additional attributes to display to the user on Identity Selection
| List of Strings
| memberof, uid, ..
| memberof,uid,org

|===
